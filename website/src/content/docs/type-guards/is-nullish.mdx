Type guard that returns `true` when the value is `null` or `undefined` and narrows the argument to `null | undefined` in the truthy branch. By symmetry, the `else` branch removes both from the type — handy for guarding optional/maybe values while preserving `0`, `""`, and `false`.

For a non-narrowing equivalent that follows the named-object parameter convention, see [`isNil`](/comparisons/is-nil/).

## Signature

```ts
function isNullish(value: unknown): value is null | undefined
```

## Parameters

| Name  | Type      | Required | Description         |
| ----- | --------- | -------- | ------------------- |
| value | `unknown` | Yes      | The value to check  |

## Returns

`boolean` (type predicate `value is null | undefined`) — `true` if the value is `null` or `undefined`, `false` otherwise.

## Examples

```ts
const value: string | null | undefined = maybeString();
if (isNullish(value)) {
  // value is narrowed to null | undefined
  return "missing";
}
// value is narrowed to string
return value.toUpperCase();

isNullish(null);       // => true
isNullish(undefined);  // => true
isNullish(0);          // => false
isNullish("");         // => false
isNullish(false);      // => false
isNullish(Number.NaN); // => false
```

## Edge Cases

- Falsy non-nullish values (`0`, `""`, `false`, `NaN`, `0n`) return `false`.
- Equivalent to the loose-equality idiom `value == null`, using strict equality for clarity.
- Same runtime behavior as [`isNil`](/comparisons/is-nil/); the difference is the type predicate, which narrows the caller's variable in both branches.

## Convention note

Type guards use a single positional argument rather than the library's named-object parameter convention so that TypeScript can narrow the caller's variable inside the `if`/`else` branches.
