Type guard that returns `true` when the value is a primitive `string` and narrows the argument to `string` in the truthy branch. Matches `typeof value === "string"` semantics — boxed `String` objects return `false`.

## Signature

```ts
function isString(value: unknown): value is string
```

## Parameters

| Name  | Type      | Required | Description         |
| ----- | --------- | -------- | ------------------- |
| value | `unknown` | Yes      | The value to check  |

## Returns

`boolean` (type predicate `value is string`) — `true` if the value is a primitive string, `false` otherwise. TypeScript uses the predicate to narrow the caller's variable.

## Examples

```ts
const value: unknown = "hello";
if (isString(value)) {
  // value is narrowed to string
  console.log(value.toUpperCase()); // "HELLO"
}

isString("");                  // => true
isString("abc");               // => true
isString(123);                 // => false
isString(null);                // => false
isString(undefined);           // => false
isString(new String("boxed")); // => false
```

## Edge Cases

- Boxed `String` objects (e.g. `new String("x")`) return `false` — they are objects, not strings.
- Empty strings (`""`) are still strings — return `true`.
- The check is `typeof value === "string"`; no coercion is performed.

## Convention note

Type guards use a single positional argument rather than the library's named-object parameter convention so that TypeScript can narrow the caller's variable inside the `if`/`else` branches. With a destructured parameter, the predicate would narrow only the local property, not the caller's identifier.
